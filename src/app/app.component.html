About the Postcoordination demonstrator

This demo complements SNOMED International's Practical Guide to Postcoordination, which is currently being developed. The front-end application showcases potential clinical user interfaces that support postcoordination. It also provides a technical overview of the postcoordinated expression generation process starting from the Close to User Form (CUF).

The postcoordinated expressions are submitted to an expression repository and transformed into the Classifiable Form (CF), which is then used to generate the Necessary Normal Form (NNF) by running a description logic classifier. The expressions stored in the repository can be queried using ECL. The processing and storage of expressions are handled by Snowstorm-X, a development version of Snowstorm that incorporates the FHIR APIs described in the postcoordination guide.

This is a reference implementation and other terminology servers are expected to implement compatible APIs to support postcoordination.
